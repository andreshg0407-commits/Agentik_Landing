## 7. Indicator Map — 12 Required Indicators

Sprint: SAG-IMPORT-RESEARCH-01

This section maps each of the 12 indicators required by the Importaciones module to their SAG data source, current implementation status, and identified issues.

### 7.1 Indicator Detail

#### Indicator 1: Fecha de ingreso (Entry date)
- **SAG Source**: `MOVIMIENTOS.d_fecha_documento` WHERE `ka_ni_fuente IN (182, 189)`
- **Status**: FOUND in SAG
- **Evidence**: All 12 sample references have FI(182) dates. Example: ref 9103 first import 2024-03-23, ref 34831-6 first import 2024-03-06.
- **Current implementation**: BUGGY — uses C1/C2 fuentes (PURCHASE_FUENTE_IDS = [1, 95]) which return ZERO rows for import products. Result: entryDate always null.
- **Correct approach**: Use FI(182) and FT(189) only. PX(184) dates are provision dates, not physical receipt dates.

#### Indicator 2: Cantidad importada (Total imported quantity)
- **SAG Source**: `SUM(MOVIMIENTOS_ITEMS.n_cantidad)` WHERE `ka_ni_fuente IN (182, 189)` AND `n_cantidad > 0`
- **Status**: FOUND in SAG
- **Evidence**: ref 9103 = 1,440 units (FI only), ref C2-AL-1 = 200 (FI) + 1,800 (FT) = 2,000 units, ref 34831-6 = 598 units (FI).
- **Current implementation**: BUGGY — same C1/C2 issue. totalImported always 0.
- **Double-counting risk**: FI and FT can document the SAME shipment (e.g., C2-AL-1 doc 37119 appears in both FI and FT on 2024-10-03). Use FI as authoritative; use FT only when no matching FI exists for that shipment.

#### Indicator 3: Cantidad restante (Remaining stock)
- **SAG Source**: `ProductInventoryLevel.quantity` WHERE `warehouseId IN import_warehouses`
- **Status**: INFERRED (from Prisma, not direct SAG query)
- **Evidence**: ProductInventoryLevel already synced from SAG inventory.
- **Current implementation**: PARTIAL — `IMPORT_WAREHOUSE_CODES = ["24", "42", "43", "44", "45", "46"]` misses ss_codigo "26" (IMPORTACION PARTE 2), "27" (IMPORTACION PARTE 1), "30" (IMPO CONTENEDOR 2).
- **Impact**: Import stock from 3 warehouses not counted. Refs 9103 (WH 36→ss26), 34831-6 (WH 37→ss27), C2-AL-1 (WH 41→ss30) all have entries in missing warehouses.

#### Indicator 4: Ventas acumuladas (Accumulated sales)
- **SAG Source**: `SUM(CustomerOrderLine.quantity)` WHERE `order.status = 'FACTURADO'` AND `quantity > 0`
- **Status**: INFERRED (from Prisma CustomerOrderLine)
- **Evidence**: Already implemented in import-service.ts. Example: ref 9103 has 671 movement rows across sales fuentes (FE, FD, FC, FG, FA, FW, PD).
- **Current implementation**: WORKING — correctly uses CustomerOrderLine with gross/returns separation.

#### Indicator 5: Ventas ultimos 6 meses (6-month sales)

### 8.3 Key Observation: FI + FT Same Shipment

FI and FT can appear on the SAME DATE with the SAME document number for the SAME products.

Evidence from C2-AL-1 on 2024-10-03:
- FI doc 37119: 200 units → WH 41
- FT doc 37119: 1,800 units → WH 41

This suggests FI = national portion, FT = international portion of the same shipment.

**Recommendation**: Use FI as the authoritative import receipt. Use FT only when no matching FI exists for that date+document combination. This avoids double-counting while still capturing China-only shipments.
